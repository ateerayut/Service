# 🎓 Zero to Hero: เริ่มต้น .NET Minimal API ฉบับจับมือทำ

ยินดีต้อนรับสู่โลกของ **.NET Minimal API**! คู่มือนี้ออกแบบมาสำหรับคนที่อาจจะไม่เคยเขียน .NET มาก่อน หรือเคยเขียนแบบ Controller มาและอยากเริ่มต้นใหม่ในแบบที่ง่ายและทันสมัยที่สุด

---

## 1. Minimal API คืออะไร? (ใน 1 ประโยค)
มันคือการสร้าง API โดยเขียนโค้ดเฉพาะที่จำเป็นจริงๆ ตัดส่วนเกิน (Boilerplate) ออก เพื่อให้ได้ประสิทธิภาพสูงสุดและเขียนง่ายที่สุด

---

## 2. เสาหลัก 3 ต้นของ Minimal API

ในการสร้าง API หนึ่งตัว คุณต้องเข้าใจ 3 เรื่องนี้:

### A. Routing (ทางเดิน)
คือการบอกว่า URL ไหน จะให้ใครทำงาน
```csharp
app.MapGet("/hello", () => "สวัสดี!");
```

### B. Parameter Binding (การรับของ)
คือการรับข้อมูลจากภายนอกเข้ามาในฟังก์ชัน
```csharp
// รับ {name} จาก URL และรับ age จาก Query String (?age=25)
app.MapGet("/hello/{name}", (string name, int age) => $"สวัสดี {name} อายุ {age}");
```

### C. Results (การส่งของคืน)
คือการส่งคำตอบกลับไปให้คนเรียก (พร้อม Status Code)
```csharp
app.MapGet("/data", () => Results.Ok(new { Id = 1, Name = "Item" })); // ส่ง 200 OK
```

---

## 3. หัวใจสำคัญ: Dependency Injection (DI)

ให้นึกว่าแอปพลิเคชันของคุณคือ **"กล่องเครื่องมือ"**
- แทนที่เราจะสร้างค้อนเองทุกครั้งที่ต้องการตอกตะปู เราจะหยิบค้อนจากกล่องเครื่องมือมาใช้
- ใน Minimal API เราสามารถหยิบ "เครื่องมือ" (Services) มาใช้ได้ง่ายๆ ผ่านพารามิเตอร์:

```csharp
app.MapGet("/products", (IProductRepository repo) => {
    // หยิบ repo มาจากกล่องเครื่องมือ (DI) แล้วใช้งานได้เลย!
    return repo.List();
});
```

---

## 4. โครงสร้างของ Template นี้ (Clean Architecture)

แม้ Minimal API จะให้เราเขียนทุกอย่างในไฟล์เดียวได้ แต่ในระดับทำงานจริง เราจะแยกเป็น 4 ชั้นเพื่อความเป็นระเบียบ:

1.  **Domain**: เก็บ "ความจริง" ของธุรกิจ (เช่น สินค้าต้องมีชื่อและราคา)
2.  **Application**: เก็บ "ขั้นตอนการทำงาน" (เรียกว่า Use Case)
3.  **Infrastructure**: เก็บ "การติดต่อข้างนอก" (เช่น การบันทึกลงฐานข้อมูล)
4.  **Api**: เก็บ "หน้าด่าน" (Endpoint) ที่คอยรับ Request

---

## 5. Workshop: ลองเพิ่ม API ใหม่ (แบบ Step-by-Step)

สมมติว่าคุณต้องการเพิ่มฟีเจอร์ **"ประกาศข่าว (News)"** ให้ทำตามนี้:

### Step 1: สร้างหัวใจ (Domain)
สร้างไฟล์ `News.cs` ในชั้น Domain เพื่อบอกว่าข่าวมีหน้าตาอย่างไร
```csharp
public record News(Guid Id, string Title, string Content);
```

### Step 2: สร้างขั้นตอน (Application)
สร้าง Use Case เพื่อบอกว่า "การดึงข่าว" ทำอย่างไร (ในชั้น Application)

### Step 3: สร้างหน้าด่าน (API)
สร้างไฟล์ `NewsEndpoints.cs` ในชั้น API และลงทะเบียนทางเดิน:
```csharp
public static void MapNewsEndpoints(this IEndpointRouteBuilder app)
{
    app.MapGet("/news", () => Results.Ok(new[] { "ข่าวเช้านี้...", "ข่าวเย็นนี้..." }));
}
```

### Step 4: ลงทะเบียนใน Program.cs
ไปที่ `Program.cs` แล้วเพิ่ม `app.MapNewsEndpoints();` เพื่อให้แอปฯ รู้จักทางเดินใหม่นี้

## 6. เคล็ดลับการเรียนรู้สำหรับ Jr. Dev

1.  **ก๊อปปี้แล้วลองแก้**: เริ่มจากดูฟีเจอร์ `Products` ที่มีอยู่แล้ว ลองเปลี่ยนชื่อฟิลด์ ลองเปลี่ยนเงื่อนไขดู
2.  **ใช้ Scalar UI ให้เป็น**: รันแอปแล้วไปที่ `/scalar/v1` เพื่อลองกดเรียก API จริงๆ ดูว่ามันส่งอะไรกลับมา
3.  **อ่าน Log**: ดูใน Console เวลาแอปฯ รัน มันจะบอกทุกลำดับที่เกิดขึ้น
